# audio backend dll
ゲームのオーディオ処理をdllにまとめる計画。
オーディオ処理は、 C++ のライブラリを使いたい。が、ゲーム本体まで C++ で開発するのはいやなので、 go で作りたい。
最終的には、単純な C API 互換の dll にすることで、 go から簡単に呼び出せるようにして、オーディオ処理は go の外に完全に任せるということを目指したい。

# 条件
シンプルな C API から呼び出せるようにする。
中身は C++ でかまわないので、 dll を呼び出す薄い層を作って、後ろ側を C++ でガンガン書く。
構造体は使っても良い。
自分のゲームで使うことを前提とするため、汎用化は考えなくてよい。
音声ファイルのロードは、メモリから行う。ファイルからは読み込まない。なぜかというと、ファイルから直接読むと、入力ファイルの柔軟性が落ちるため（暗号化できないなど）
基本的に、 fmod のラッパーを作る。
基本的に、全ての関数はnonblockingで、呼び出したらすぐに戻る。裏で自動で処理が行われる。
audio_vr*** の関数は、 google resonance audio のプラグインを利用して spatial audio を出力する。

## ツールチェーン
Visual Studio ツールチェーンを利用する。
AI エージェントは、 x64 開発者用コマンドプロンプトの上で実行されていると仮定してよい。つまり、 MSVC のコマンドが通る前提で作業を行う。

## fmod API について
この dll は、 FMOD soundsystem のラッパーとして動作します。
基本的に、 FMOD の core API (low level APIとも言います) を使用してください。
FMOD のリファレンスを、このリポジトリの fmod_manual に配置しているので、必要に応じて参照しながら実装してください。
