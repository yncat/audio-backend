バグの調査と修正をしてください。
バグレポートの内容: $ARGUMENTS

作業手順:
1. この後に示す「ヒント」を参考に、バグが発生していそうなファイルを特定します。
2. コードを調査して、バグを発見します。
3. バグを修正します。
4. /build SlashCommand を使用してビルドを実行します。
5. バグが治ったかどうかのテストをユーザーに依頼します。
6. バグが治ったと入力されたら、以下の内容で修正レポートを作成します。
- 現在の commit hash
- バグレポートの内容
- 原因
- 修正内容
修正レポートを docs/bugreports のディレクトリに markdown で書き込みます。この際に、短くて適切な英語のファイル名を付けて、新規ファイルとして保存します。
